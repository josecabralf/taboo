    taboo

    Run coding agents in a sandbox. Land their commits on your branch.

    taboo is a Go library and CLI. It runs AI coding agents inside isolated Canonical workshops and lands their commits on a fresh branch. Nothing touches your working tree.

    Three steps from prompt to commit

    You describe the work in Go. taboo handles the sandbox, the agent loop, and the fresh branch. The agent commits in place on that branch; your working tree is never touched.

    1. 01

      Define the run

      Name a workflow in taboo.yaml — its agent, prompt, and limits. The config is the contract.

      taboo.yaml
      workflows:
  implement:
    agent: claude-code
    prompt: Implement the issue.
    2. 02

      Run it in isolation

      taboo starts a fresh Canonical workshop and runs the agent inside it.

      terminal
      $ taboo run implement
✓ workshop  ubuntu 24.04
◐ agent     plan → edit → test
◐ verify    go test ./...
  running in sandbox…
    3. 03

      Land verified commits

      After the agent commits, the branch ref lands in your repo — visible to git right away.

      git log
      ● a1c39f2  (taboo/implement)
│ http: add backoff
│ retries to client
│ 2 files · +38 −4 ✓
● e7b1a0c  main

    Go library

    Call taboo from your own code

    Run agents from CI jobs, internal platforms, or batch refactors. One call resolves a workflow and runs it; a commit comes back.

    Library reference→
    import
    import "github.com/josecabralf/taboo"

res, _ := taboo.RunWorkflow(ctx, ".", "implement",
    vars, taboo.PlanOverrides{}, taboo.NewExecCommander())
log.Printf("landed %s", res.Commit)
    Command line

    Run it from the terminal

    No Go required. Scaffold a project and run it with two commands.

    $ taboo init && taboo run

    Why taboo

    Isolated by default

    Every agent runs in a fresh Canonical workshop: an Ubuntu sandbox with your repo and toolchain, and nothing else from your host. If a run goes wrong, the damage stops at the sandbox.

    Commits, not loose diffs

    The agent commits its work in place on a fresh branch, so you review real git history with proper commits, not a pile of unstaged edits.

    Contract-first and deterministic

    The taboo.yaml workflow is the spec, read the same way by the CLI and Go callers. The same workflow and the same inputs produce the same run, whether you start it from your editor or from CI.
