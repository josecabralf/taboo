taboo

Run coding agents in a sandbox. Land their commits on your branch.

taboo is a Go library and CLI. It runs AI coding agents inside isolated Canonical workshops, then merges their commits onto your branch. Nothing touches your working tree until the tests pass.

$ go get github.com/josecabralf/taboo
Read the quickstart→ View on GitHub
main.go 
result, err := taboo.RunWorkflow(ctx, taboo.Workflow{
    Agent:  taboo.Claude,
    Branch: "main",
    Task:   "Add retries to the HTTP client",
})
// result.Commits are now on your branch

Three steps from prompt to commit

You describe the work in Go. taboo handles the sandbox, the agent loop, and the merge. If the work fails, nothing reaches your branch.

  1. 01

    Define the run

    Describe the agent, the branch, and the task. The Go types are the contract.

    workflow.go
    wf := taboo.Workflow{
    Agent:  taboo.Claude,
    Branch: "main",
    Task:   "Add retries",
}
  2. 02

    Run it in isolation

    taboo starts a fresh Canonical workshop and runs the agent inside it.

    terminal
    $ taboo run ./workflow
✓ workshop  ubuntu 24.04
◐ agent     plan → edit → test
◐ verify    go test ./...
  running in sandbox…
  3. 03

    Land verified commits

    After the tests pass, taboo merges the commits onto your branch.

    git log
    ● a1c39f2  (main)
│ http: add backoff
│ retries to client
│ 2 files · +38 −4 ✓
● e7b1a0c  main

Why taboo

Isolated by default

Every agent runs in a fresh Canonical workshop: an Ubuntu sandbox with your repo and toolchain, and nothing else from your host. If a run goes wrong, the damage stops at the sandbox.

Commits, not loose diffs

taboo runs the tests, then merges commits onto your branch. You review git history with passing tests, not a pile of unstaged edits.

Contract-first and deterministic

The Go types are the spec. The same workflow and the same inputs produce the same run, whether you start it from your editor or from CI.
